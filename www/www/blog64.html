64-bit version of your Xamarin Native or Xamarin Forms Android App

Did you also receive the pesky email from Google Play, asking you to create 64-bit version of your fleet of apps? You received instructions to fix the issue, but you realize you don’t use Android Studio for development. So here’s a quick guide for the business analysts, testers and developers, on all you need to know:

Problem Summary

Deadline

You obviously don’t want to wait until August 1, 2019 when your app can’t be updated for the store, so it’s definitely smart to start now.

By August 1, 2019, all apps that use native code must provide a 64-bit version in addition to the 32-bit version in order to publish an update.

But the Email says “apps that use native code”?

You read that right, the change is only needed for native code. Since Xamarin Forms and Xamarin Native both compile down to native code, so this applies to all the Xamarin apps you have published, or are going to publish.

But Why?

To keep up with the times

64-bit CPUs deliver faster, richer experiences for your users. Adding a 64-bit version of your app provides performance improvements, makes way for future innovation, and sets you up for devices with 64-bit only hardware.

Qualifications Required

You really don’t need to know how to code. Although there are many ways of doing this, I will walk you through the simplest one, which will basically involve a couple of clicks and scrolls.

Software Needed

You can use either a Mac or a PC. You need to install the latest version of MS Visual Studio, available for Mac and PC, and ensure that you also install the Xamarin Framework. You will also need the Keystore used for the published app, and the associated keystore alias and keystore password. For more information on the keystore, read the “Problems you can run into” section.

The Fix

Making the changes

Make sure you have the latest version of your app, as you don’t want to publish an older version than what’s published. Open the {AppName}.sln solution file in Visual Studio that should contain your {AppName}.Droid project. Then follow the steps shown in the pictures:

  1. Double Click on the Droid project in the solution.

2. This should open your Project Options. Then Select Android Build to change your Build settings.

3. Click on the Advanced tab and make sure that “arm64-v8a” and “x86_64” Supported ABIs are selected.

4. Change the configuration to Release (and all other configurations you use) and make sure that the “arm64-v8a” and “x86_64” ABIs are supported.

5. Press OK and you are done.

6. In order to publish the app now, make sure you set the target to your Android project in the release configuration.

7. Go to the build Menu, and select Rebuild All to make sure the solution is cache is clean.

8. Once that’s done, Right click on your Droid project.

9. Select the Archive for Publishing option and wait for it to build your Android project APK Archive.

And then you can just follow the instructions given here from Microsoft, but just “Import an Existing Key” instead.

Problems you can run into

  1. Often times the Keystore file (xxx.keystore) for an app is lost. Or the previous developer does not remember the Keystore Alias and Keystore Password (like a username and password for the app). In such cases, you can’t really do anything, and you usually have to create a brand new app listing. It’s happened to me before.

Easy to Add, Free Enhancements for Developers

  1. Curious to know what your users are doing in the app or how long they are spending? With the MS App Center Analytics, you can track people’s actions and page views using one line statements like Analytics.TrackEvent(“PageView: HomePage”); or Analytics.TrackEvent(“SelectAction: Logout”);
  2. Adding Push Notifications doesn’t have to be so difficult. With the MS App Center Push Notifications, you get an easy to use portal where you can instantly send push notifications to all your users instantly. As shown in the Github linked above, you can also use the api.appcenter.ms to get a list of notifications published if you want to add an additional screen.
