WiFi Build and Debug with Android 11 on Xamarin, Flutter & React Native

No more cables on your tables

Wireless build and debug for Android is a feature cross-platform mobile developers have been waiting for years, especially since iOS enabled wireless debugging years ago. From Android 11 onwards, the process has been simplified and you can finally get rid of wired build and debugging!

There’s a few steps involved, but luckily you just have to do it once per wireless connection, and you’re good across all your apps. If you are already using Android SDK 30, you should be fine. Or else you need to install SDK 30 on your computer .

Steps to Wireless Build and Debugging with Android

  1. On your Android 11 device, go to your Settings -> Tap on Advanced and go to the Developer options.
  2. Then tap on Wireless Debugging switch, and always allow wireless debugging for your personal wireless network in the popup dialog
  3. Then tap on the Wireless Debugging option and select “Pair device with pairing code” and it will give you a 6 digit code and IP address as you can see in the image
  4. Type the IP address and port from your phone in the "Pair with device" pop-up screen above into your computer terminal: "adb pair {ipaddress}:{port}". Xamarin devs can access adb inside Visual Studio -> Tools -> SDK Command Prompt
  5. Then take the IP Address and port from the last screen and enter the following command "adb connect {ipaddress}:{port}".

Voilà, say goodbye to your Android cables, while selecting the same device from the same drop down that you have always used. While figuring this out, my heart skipped a beat while I took out the wire from my Mac. Goodbye weak cable that stopped debugging if I tilted the phone too much, I will not miss you.

Final Thoughts

As you will notice, the difference between iOS & Android wireless debugging is that iOS requires you to configure your device for each app, where as Android requires you to configure your device for each wireless network. If you’re using Android 10 or below, you can try to follow this tutorial. If you have any questions, feel free to message me and don’t forget to check out my other articles.
